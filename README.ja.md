# 🍎 Open Vocabularies (オープンボキャブラリー)

主要な語学検定（TOEIC、JLPT、HSKなど）のためのオープンソースでコミュニティ主導の語彙データセット。私たちの使命は、高品質なベースワードリストを取り込み、コミュニティの貢献によって洗練されたマルチロケールなデータセットに変換することです。これらのデータセットは、[Ringoo](https://ringoo.ai)アプリの語彙機能で使用されています。

## 🚀 目的

このリポジトリの使命は、言語学習者に**非常に正確で高品質、かつ機械読み取り可能**な単語リストを提供することです。これらのリストをオープンソース化することで、以下を目指しています：

1. **正確性の確保:** 教師やネイティブスピーカーが用語を監査し、修正できるようにします。
2. **標準化:** 言語データに対して一貫したJSONスキーマを提供します。
3. **アクセシビリティ:** Anki、CSVなどの一般的なフォーマットへのエクスポートをサポートします。
4. **マルチロケール対応:** あらゆる背景を持つ学習者を支援するため、高品質な翻訳とローカライズ版を提供します。

---

## 📂 リポジトリ構造

データは言語と試験タイプごとに `data/` ディレクトリ配下に整理されています：

```text
open-vocabularies/
├── data/                # 語彙データセット
│   ├── ja/              # 日本語試験データセット
│   │   ├── jlpt-n5.json         # マスターJSONデータセット
│   │   ├── jlpt-n5.csv          # アップストリームソース（元データ）
│   │   ├── jlpt-n5_{locale}.csv # ローカライズ済みデータセット（配布用）
│   │   └── ...
│   └── en/              # 英語試験データセット
│       ├── toeic.json           # マスターJSONデータセット
│       ├── toeic.csv            # アップストリームソース（元データ）
│       └── toeic_{locale}.csv   # ローカライズ済みデータセット（配布用）
├── schema.json          # マスターデータ構造定義
└── scripts/             # データバリデーションおよびエクスポートツール
```

## 📜 出典とクレジット

私たちは先人の知見に基づいています。ベースとなるデータセットは高品質な教育リソースから派生し、コミュニティによって洗練されています：

- **TOEIC語彙:** [Pass the TOEIC Test](https://www.pass-the-toeic-test.com/toeic-word-list.php) から提供されたベースリストに基づいています。
- **JLPT語彙:** [Open Anki JLPT Decks](https://github.com/jamsinclair/open-anki-jlpt-decks) から提供されたベースリストに基づいています。
- **メンテナンス:** これらのリストは、現代の用法を反映し、試験特有のニュアンスを補完するために、**Ringooアプリのユーザー**およびコミュニティの貢献者によって積極的にレビュー、修正、拡張されています。

## 🛠 データフォーマット

豊富なメタデータと使いやすさのバランスをとるため、**JSON**と**CSV**の両方の形式でデータセットを提供しています。

### CSV (配布用)
迅速なインポートと基本的な学習のためのAnki互換フォーマット。

```csv
expression,reading,meaning,tags,guid
innovation,,a new method, idea, or product,TOEIC,unique-id-001
```

---

## 🗃 エクスポートと使用方法

`data/` ディレクトリに事前生成されたCSVを提供していますが、スクリプトを使用してカスタムエクスポートを生成することもできます：
1. リポジトリをクローンする。
2. `npm run export:anki` を実行する。
3. 生成された `.csv` または `.apkg` ファイルをAnkiアプリにインポートする。

---

## 🤝 貢献方法

貢献を歓迎します！誤字脱字、誤訳の発見、または新しい試験リストの追加を希望される場合は：
1. リポジトリを**フォーク**する。
2. 変更を加え、それが `schema.json` に適合していることを確認する。
3. 変更内容の簡単な説明を添えて **プルリクエストを送信** する。
